How to add web maps to your portfolio website?


    Step 1. Define height and margins

    Our portfolio webpage has a title bar, which may hide the top portion of the maps and cause inproper display. So let's fix that by adding margins first. In addition, we will also add a fixed height for the map containers.

  • Find your portfolio folder (e.g., ges387). Edit the index.html file, copy the following lines and paste them inside the head section: 
    <style>
    body {
        height: 700px;
    }

    .item{
      margin-left:-10px;
      margin-right:-10px;
      height:700px;
    }

    @media (min-width: 1200px) {
      .item{
        margin-top: 40px;
      }
      .carousel-caption{
        bottom: 70px;
      }
    }

    @media (min-width: 992px) and (max-width: 1199px) {
      .item{
        margin-top: 90px;
      }
      .carousel-caption{
        bottom: 120px;
      }
    }

    @media (min-width: 768px) and (max-width: 991px){
      .item{
        margin-top: 90px;
      }
      .carousel-caption{
        bottom: 120px;
      }
    }

    @media (min-width: 488px) and (max-width: 767px) {
      .item{
        margin-top: 40px;
      }
      .carousel-caption{
        bottom: 70px;
      }
    }

    @media (max-width: 487px){
      .item{
        margin-top: 90px;
      }
      .carousel-caption{
        bottom: 120px;
      }
    }
</style>
    Specifically, we have added fixed height for the map containers to be 700px. We have also adjusted the top margin of the web maps to be added and the bottom margin of the captions.

    The lengthy code defines margins that will also automatically adjust based on browser window size so that your website can be displayed properly when viewed from tablet and/or smart phones (mobile-friendly).

    • Step 4. Sync/publish the changes to GitHub

  • If you are not familiar with GitHub, it is recommended to make a copy of your portfolio folder for backup.
  • Launch GitHub Desktop. Click the button on the upper right corner and select Options to log in with your account.

  • If you see your portfolio repository (e.g., ges387) listed on the left side, right-click it and select Open in Explorer to make sure it links to the right folder (very important!!).

  • No worries if you do not see the repository, click the button on the upper left corner and select Add and browse to your portfolio folder to add the repository.

  • If you are sure the repository is set up correctly, select the ges387 repository and click the Changes tab (see image below). If you have made some changes since last Sync/Publish, you should see a solid circle next to the Changes which indicates changes have been made in the local path. You may also click on each file to double-check if they reflect the most recent changes you have made.

  • Make sure that gh-pages is the current branch. Enter some information for Summary (It can be anything, the idea is to remind you what kind of changes you have just made). The Description is optional. Then click Commit to gh-pages.

  • If the Commit was successful, it would show No local changes. Then, click Sync to synchronize the changes.

  • Open your web browser, go to your portfolio website to check if it worked (ask me if you don't remember your url). Note you can make as many changes as you want this way (Login - Connect the folder - Commit - Sync).
